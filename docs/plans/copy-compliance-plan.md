# Copy Compliance Plan

## Goal

Public PRSLOY copy should sell a closed beta privacy/connectivity product without advertising access to blocked resources or publishing technical bypass instructions.

## Priority 1

1. Replace public "VPN without blocks" language with "private/protected access by invite".
2. Replace "buy", "payment", "crypto only", "no email" language with beta invite/access language.
3. Remove public "bypass", "whitelist", "provider sees", "entry/exit" phrasing from the landing page and FAQ previews.
4. Keep the $5 price as a beta access reference, not as an active checkout promise.
5. Make pricing page match the current no-payment reality: request invite, verify capacity, manual issue.

## Priority 2

1. Move client/protocol/setup details out of public hero/pricing copy and into dashboard/setup context.
2. Soften infrastructure claims from automatic guarantees to managed-route language.
3. Make invite-only sound like quality control, not scarcity for scarcity's sake.
4. Make FAQ investor-safe: direct, honest, no slang, no overclaiming.

## Priority 3

1. Replace time-sensitive "week 1 / day 1" labels with stable "private beta" labels.
2. Reduce aggressive closing CTAs.
3. Keep terminal/mono style for labels, but let product meaning stay plain.
